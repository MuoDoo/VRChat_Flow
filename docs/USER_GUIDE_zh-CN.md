# VRCFlow 使用指南

中文 | [English](USER_GUIDE.md) | [日本語](USER_GUIDE_ja.md)

VRCFlow 是一款 VRChat 实时语音翻译应用。对麦克风说话，语音会被自动识别、翻译，并显示在 VRChat 的 Chatbox 中。

## 下载安装（Windows）

1. 前往 [Releases 页面](https://github.com/MuoDoo/VRCFlow/releases)
2. 下载最新的 `.exe` 安装包
3. 运行安装程序，选择安装路径，完成安装
4. 从开始菜单或桌面快捷方式启动 **VRCFlow**

> 安装包适用于 Windows x64（Windows 10/11），无需额外运行环境。

## 快速上手

### 1. 选择 Provider（语音服务商）

VRCFlow 支持多个语音识别/翻译 Provider。打开 **设置**（右上角）并切换到 **Provider** 标签页进行选择。

#### 阿里云 DashScope（百炼）

内置的原始 Provider。使用阿里云百炼的实时 WebSocket API，同步完成语音识别和翻译。

**模型：gummy-chat-v1**

| 特性 | 详情 |
|------|------|
| **协议** | WebSocket 双工流式传输（实时） |
| **ASR + 翻译** | 一次调用同时完成语音识别和翻译 |
| **支持语言** | 中文、英语、日语、韩语 |
| **计费** | ¥0.00015/秒/通道，双通道计费（ASR + 翻译 = 2× 时长） |
| **免费额度** | 新用户赠送 10 小时免费额度 |
| **计费方式** | 仅对实际说话时长计费 |

**优势：**
- 超低延迟 — 实时流式处理，说话的同时就能看到结果
- 针对 VRChat 对话中的短语音片段进行了优化
- 计费简单，无需计算 token 消耗
- 中文 ↔ 其他语言翻译性价比极高

**如何获取 API Key：**
1. 访问 [阿里云百炼 API Key 页面](https://bailian.console.aliyun.com/cn-beijing/?apiKey=1&tab=model#/api-key)
2. 如果没有账号，先注册一个（新用户有 10 小时免费额度）
3. 创建并复制你的 API Key

#### OpenRouter

可选的替代 Provider，通过统一 API 路由到多种 AI 模型。提供灵活的模型选择和透明的定价。

**如何获取 API Key：**
1. 访问 [OpenRouter Keys](https://openrouter.ai/keys)
2. 创建账号并生成 API Key
3. 充值账户余额

##### 模型：Voxtral Small 24B（推荐）

Mistral 的音频能力模型，具备顶尖的语音转录和翻译能力。

| 特性 | 详情 |
|------|------|
| **提供商** | Mistral AI（通过 OpenRouter） |
| **协议** | REST API + Function Calling |
| **上下文窗口** | 32K tokens |
| **最大输出** | 8,192 tokens |
| **音频输入计费** | $100 / 1M 秒（约 $0.006/分钟） |
| **文本输入计费** | $0.10 / 1M tokens |
| **文本输出计费** | $0.30 / 1M tokens |
| **音频格式** | WAV、MP3、FLAC、Opus、OGG |

**优势：**
- 业界顶尖的语音转录和翻译准确度
- 短音频片段成本极低（非常适合 VRChat 场景）
- 保持了出色的文本理解能力
- 支持 Function Calling，结构化输出更可靠

##### 模型：Gemini 3.1 Flash Lite

Google 的高效模型，为大批量、高性价比场景优化。

| 特性 | 详情 |
|------|------|
| **提供商** | Google（通过 OpenRouter） |
| **协议** | REST API + Function Calling |
| **上下文窗口** | 1M tokens |
| **最大输出** | 65,536 tokens |
| **音频输入计费** | $0.50 / 1M tokens |
| **文本输入计费** | $0.25 / 1M tokens |
| **文本输出计费** | $1.50 / 1M tokens |
| **音频格式** | WAV、MP3、FLAC、Opus、OGG |

**优势：**
- 超大 1M token 上下文窗口
- 支持文本、图片、视频、文件和音频输入
- 隐式缓存支持（缓存读取：$0.025/1M tokens）
- 成本仅为 Gemini 3 Flash 的一半

### Provider 对比

| | DashScope | OpenRouter (Voxtral) | OpenRouter (Gemini) |
|---|---|---|---|
| **最适合** | 中文 ↔ 其他语言，最低延迟 | 最高准确度，短音频 | 大批量，预算敏感 |
| **延迟** | 实时流式 | REST（一次性） | REST（一次性） |
| **计费模式** | 按秒计费 | 按秒 + 按 token | 按 token |
| **每分钟约** | ~¥0.018（约 $0.0025） | ~$0.006 | 取决于音频长度 |
| **免费额度** | 10 小时 | 无 | 无 |
| **模型选择** | 固定（gummy-chat-v1） | 可选 | 可选 |

### 2. 配置设置

点击右上角的 **设置** 按钮打开设置面板：

**Provider 标签页：**

| 设置项 | 说明 |
|--------|------|
| **Provider** | 选择 DashScope 或 OpenRouter |
| **API Key** | 所选 Provider 的 API 密钥 |
| **模型** | 模型选择（仅 OpenRouter） |

**通用标签页：**

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| **源语言** | 你说的语言 | 中文 |
| **目标语言** | 翻译成的语言 | 英语 |
| **OSC 端口** | VRChat OSC 的 UDP 端口 | `9000` |
| **SteamVR 悬浮窗** | 启用/禁用 VR 悬浮窗显示 | 关闭 |
| **处理超时** | 等待 API 响应的最大秒数 | `5` |
| **语音填充** | 语音结束检测前的静默填充（毫秒） | `600` |
| **显示货币** | 费用显示货币（CNY/USD/JPY） | CNY |

可选语言：中文、英语、日语、韩语。

### 3. 切换界面语言

使用右上角的语言下拉菜单，可切换界面为 **English**、**中文** 或 **日本語**。切换后自动保存。

## 使用方法

### 麦克风翻译

1. 点击屏幕底部的绿色 **开始** 按钮
2. 系统弹出麦克风权限请求时，点击允许
3. 状态显示 **"正在聆听..."** — 正常说话即可
4. 停顿后，VRCFlow 自动检测到语音结束，进行识别和翻译
5. 翻译结果以卡片形式显示在主区域：
   - 时间戳
   - 原文（你说的内容）
   - 译文（翻译后的内容）
   - Provider 和处理信息
6. 译文会同时通过 OSC 发送到 VRChat 的 Chatbox

> 只有在说话时才会计费。即使点击了"开始"，只要不说话就不会产生费用。

### 系统音频翻译

VRCFlow 还可以捕获和翻译系统音频（例如 VRChat 中其他人的语音）：

1. 点击底部的 **扬声器** 标签页
2. 点击 **开始** 开始捕获系统音频
3. 应用会捕获桌面音频，应用 VAD 检测，并翻译检测到的语音
4. 结果和麦克风翻译一起显示，带有扬声器图标标识

> 此功能适用于翻译语音聊天中其他人说的话。

### SteamVR 悬浮窗

在设置 → 通用 → SteamVR 悬浮窗中启用，可在 VR 中以悬浮面板形式显示翻译结果：

- 在 VR 视野中显示最近的翻译
- 消息在 8 秒后自动消失
- 最多同时显示 3 条消息

### 停止翻译

点击红色 **停止** 按钮停止监听。

### 状态指示

| 状态 | 含义 |
|------|------|
| **就绪** | 空闲状态，按「开始」启动 |
| **正在聆听...** | 麦克风/扬声器已开启，等待语音输入 |
| **处理中...** | 检测到语音，正在识别和翻译 |
| 绿色圆点 | 正在监听 |
| 橙色圆点 | 正在处理语音片段 |

### 仪表盘

点击顶栏的柱状图图标查看用量统计：
- 按 Provider 分类的费用明细（DashScope 以 CNY 显示，OpenRouter 以 USD 显示）
- 今日音频秒数、计费秒数、预估费用
- 杂音检测统计（咳嗽、环境音等）
- 每日消耗明细表，含各 Provider 详情
- 货币转换支持（CNY / USD / JPY）

### 历史记录

点击顶栏的时钟图标浏览所有历史翻译记录，包含时间戳、消耗时长、Provider 信息和 token 用量。

## VRChat 配置

在 VRChat 中显示翻译结果：

1. 打开 VRChat
2. 打开 **Action Menu**（桌面端按 R 键）
3. 进入 **Options** → **OSC** → **Enable**
4. 确认 VRCFlow 中 OSC 端口设置为 **9000**（VRChat 默认端口）
5. 翻译结果将显示在你头像上方的 Chatbox 中

> VRChat 和 VRCFlow 在同一台电脑上运行时无需额外配置。

## 常见问题

### 麦克风无法使用

- **Windows**：进入 设置 → 隐私与安全 → 麦克风，确保已开启应用访问权限
- 在 Windows 声音设置中确认麦克风已设为默认录音设备
- 尝试关闭其他可能占用麦克风的程序

### VRChat 中看不到翻译结果

- 确认 VRChat 已启用 OSC（Action Menu → Options → OSC → Enable）
- 确认 VRCFlow 的 OSC 端口与 VRChat 一致（默认 9000）
- 如果是首次启用 OSC，尝试重启 VRChat

### 显示"Loading VAD..."

语音检测模型正在加载，首次启动需几秒钟。如果长时间卡住，请重启应用。

### 识别失败

- 检查设置中的 API Key 是否正确（确保与所选 Provider 匹配）
- 检查网络连接
- **DashScope**：API 额度可能已用完 — 请前往 [百炼控制台](https://bailian.console.aliyun.com/) 查看
- **OpenRouter**：检查账户余额 — 请前往 [OpenRouter](https://openrouter.ai/credits) 查看

### 延迟高或超时

- 尝试在设置 → 通用 → 高级中减小 **语音填充** 的值
- 尝试切换其他 Provider 或模型
- 检查网络连接稳定性
